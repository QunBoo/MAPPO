1. 当前项目中IoTD 和 DAG 之间没有显式关联，传输计算中的 IoTD 是动态选取的，非固定绑定
在 _calc_T_trans()（第 486–550 行），所有非本地卸载路径都调用：

python
iotd_p = self._nearest_iotd(m)   # 第491、503、527行
而 _nearest_iotd() 的实现（第 586–590 行）是每次实时计算最近距离：

python
def _nearest_iotd(self, m: int) -> np.ndarray:
    uav_xy = self.uav_pos[m, :2]
    dists  = np.linalg.norm(self.iotd_pos[:, :2] - uav_xy, axis=1)
    return self.iotd_pos[int(np.argmin(dists))]  # 返回当前最近的那个
这意味着：

同一个 UAV 的不同任务可能关联到不同的 IoTD（因为 UAV 在移动）
不存在固定的 Task → IoTD 映射关系
IoTD 只是作为信道计算的几何参考点

然而实际上，IoTD应为实际的数据源和任务源，当涉及到任务传输计算时，应是IoTD产生DAG-->UAV合并DAG-->智能体处理DAG

目前DAG由UAV产生，导致IoTD在仿真中作用模糊；

DAG应由IoTD产生，并在UAV处或LEO卫星（IoTD直接接入LEO卫星的情况）处进行多个IoTD产生的DAG的合并。
